JSX

After learning all the hooks and nooks of html, basic css and javascript, moving forward I heard about JSX. JSX as we know is JavaScript XML. A React Component renders everything and displays it in the browser. Up until recently, we could just return HTML, but with the advent of JSX, we can now also return HTML mixed with JavaScript. Due to internal optimizations, it is quicker than standard JavaScript. However we are not required to use JSX, but it makes easier to write React applications.
Some JSX benefits include:

  • The language makes writing complex chunks of code simpler and easier.
  • Because JSX is statically coded and generally type-safe, it facilitates React to provide more helpful error messages.
  • If you are already comfortable with HTML, JSX will come naturally to you as well.
  • It is faster than standard JavaScript because it makes optimizations while it converts to standard JavaScript.


Now for the main part What is React?

React is a JavaScript library build in 2013 by the Facebook development team to make user interfaces more modular (or reusable) and easier to maintain. According to React's website, it is used to "Build encapsulated components that manage their own state, then compose them to make complex UIs." When you start dealing with React for the very first time, its official Documentation will be the best option for learning and support.


Setting Up React on your workspace

The best way to begin learning React and creating a new single-page application in React is with create-react-app.It sets up your development environment so that you can use the latest JavaScript features, provides a nice developer experience, and optimizes your app for production.

With this you can also create basic react applications without having to code much.
